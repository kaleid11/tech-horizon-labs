# Acronis Cyber Protect Cloud: Generative AI Usage Monitoring

262  viewsLast Updated:  10 Feb 2026Article Number:000030583

Acronis Cyber Protect Cloud

### Symptoms

You want to understand how **Generative AI usage monitoring** works in **Acronis Cyber Protect Cloud** and how Generative AI applications can be monitored and controlled.

### Environment

-   Acronis Cyber Protect Cloud
-   **Enabled services:**
    -   URL Filtering
    -   EDR/XDR
-   **Operating System:**
    -   Windows only
    -   Not supported on Windows ARM

### Solution

**Generative AI usage monitoring** is a security and compliance feature that allows organizations to:

-   Monitor usage of supported Generative AI applications
-   Control and enforce secure AI usage policies
-   Log AI-related incidents in a dedicated events log
-   View AI usage trends, summaries, and analytics through built-in reports and dashboard widgets

This feature strengthens compliance posture and reduces the risk of sensitive, confidential, or regulated data being leaked through Generative AI applications.

To enable Generative AI security:

1.  Open the **Cyber Protect console**
2.  Go to **Management > Protection plans**
3.  Select an existing protection plan and click **Edit** in the right sidebar or create a new protection plan
4.  In the protection plan sidebar, enable the **Generative AI security** module by toggling the switch next to the module name
5.  Save the protection plan

Once enabled, AI monitoring, logging, and reporting become active on the protected workloads.

Please keep in mind, that this feature is currently in the Early **Access Program** (*EAP*) status and  only available only for supported licenses (*EDR+URL filtering*).

You can find more information about this feature in our User Guide: [Generative AI security settings](https://www.acronis.com/en/support/documentation/CyberProtectionService/#generative_ai-security_settings.html)
